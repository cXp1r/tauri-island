# 设置页面 UI 美化与优化

## 改进概览

### 1. 视觉设计增强

#### 标题与徽章
- 标题添加渐变色效果（#e9f0ff → #b8d4ff）
- 字号从 28px 增至 30px，字间距优化为 -0.3px
- 徽章增强：更明显的边框、阴影和内边距

#### 卡片系统
- 增强玻璃态效果：backdrop-filter 从 blur(14px) 升级到 blur(20px) + saturate(1.2)
- 添加悬停动画：上移 2px + 阴影增强
- 卡片标题添加渐变色装饰条（3px 宽）
- 优化内边距和间距（14px → 18px）

#### 输入控件
- 边框从 1px 增至 1.5px，提升视觉清晰度
- 聚焦状态优化：
  - 阴影从 3px 增至 3px + 额外的 4px 模糊阴影
  - 添加 translateY(-1px) 微动效
- 悬停状态：边框颜色加深 + 背景完全不透明

#### 开关按钮
- 尺寸从 46x26px 增至 50x28px
- 滑块添加渐变色（#ffffff → #f0f4ff）
- 开启状态添加发光效果（box-shadow）
- 动画曲线优化为 cubic-bezier(0.34, 1.56, 0.64, 1)

#### 按钮系统
- 所有按钮添加伪元素渐变遮罩（hover 时显示）
- 主按钮阴影增强：从 0.3 增至 0.4 透明度
- 次要按钮悬停时上移 1px + 阴影增强
- 保存按钮：
  - 字号从 14px 增至 15px
  - 内边距从 12x16px 增至 14x20px
  - 添加字重 700 和字间距 0.3px

### 2. 交互体验优化

#### 动画流畅度
- 所有过渡效果统一使用 cubic-bezier(0.4, 0, 0.2, 1)
- 快捷键录制状态添加脉冲动画（pulse）
- 按钮点击反馈：translateY(1px) + scale(0.98)

#### 视觉反馈
- 卡片悬停时显示渐变遮罩
- 输入框聚焦时轻微上移
- 所有可交互元素添加明确的悬停状态

### 3. 窗口调整大小功能

#### 调整手柄
- 8 个方向的调整手柄（上下左右 + 四个角）
- 手柄尺寸：边缘 8px，角落 16px
- 自动光标样式（ns-resize, ew-resize, nwse-resize, nesw-resize）

#### 调整逻辑
- 最小尺寸限制：600x400px
- 支持从任意边缘和角落拖动
- 实时更新窗口尺寸和位置
- 错误处理和日志记录

#### Rust 配置更新
- 初始尺寸从 560x760 增至 760x840
- 添加最小尺寸限制：600x400
- 启用 resizable(true)

### 4. 布局优化

#### 间距系统
- 主容器内边距：24px → 28px
- 卡片间距：14px → 16px
- 行间距：10px → 12px
- 操作按钮间距：8px → 10px

#### 响应式设计
- 保持原有的移动端适配（@media max-width: 560px）
- 优化小屏幕下的按钮和输入框尺寸

### 5. 细节打磨

#### 文本优化
- 所有字号微调（+0.5px 到 +1.5px）
- 行高统一优化（1.45 → 1.5, 1.5 → 1.6）
- 代码块字体大小从 12px 增至 12.5px
- 添加 user-select: text 到代码块

#### 颜色对比度
- 静音文本颜色保持 #93a4c8
- 卡片标题颜色从 #8ea2c8 调整为 #7a95c0
- 状态文本字重增至 500

## 技术实现

### 前端（TypeScript）
- 导入 `getCurrentWindow` API
- 实现鼠标事件监听（mousedown, mousemove, mouseup）
- 计算窗口尺寸和位置变化
- 调用 Tauri API 更新窗口

### 后端（Rust）
- 修改两处 WebviewWindowBuilder 配置
- 设置初始尺寸和最小尺寸
- 启用窗口可调整大小

## 使用说明

1. 窗口边缘和角落现在可以拖动调整大小
2. 最小尺寸为 600x400，防止窗口过小
3. 所有交互元素都有明确的视觉反馈
4. 保持了原有的深色主题和渐变效果

## 兼容性

- 保持与原有功能的完全兼容
- 不影响现有的设置保存和加载逻辑
- 响应式设计确保在不同屏幕尺寸下正常工作
